## Overview

Customize your Janek Unon documentation space to match your brand, control user access, and export content efficiently. Start with branding to apply your colors and logo, then set permissions for team members. Finally, explore export options for sharing or backups.

<Callout kind="tip">
  Save changes after each section to apply updates immediately.
</Callout>

## Brand Customization

Apply your brand color (`#3B82F6`) and logo through the settings panel. You configure these options via the UI or a configuration file.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to **Space Settings** > **Branding**.
  </Step>
  <Step title="Set Brand Color" icon="palette">
    Enter your hex code in the **Primary Color** field.
  </Step>
  <Step title="Upload Logo" icon="image">
    Drag and drop your logo (PNG or SVG, `<500KB`).
  </Step>
  <Step title="Preview and Save" icon="check-circle">
    Review changes and click **Apply Branding**.
  </Step>
</Steps>

<Tabs>
  <Tab title="UI Config" icon="monitor">
    Use the visual editor for quick updates.
  </Tab>
  <Tab title="Config File" icon="file">
    Edit `space-config.json` directly for version control.

    <CodeGroup tabs="JSON,YAML">
```json
{
  "branding": {
    "primaryColor": "#3B82F6",
    "logoUrl": "https://your-domain.com/logo.svg",
    "fontFamily": "Inter"
  }
}
```
```yaml
branding:
  primaryColor: "#3B82F6"
  logoUrl: "https://your-domain.com/logo.svg"
  fontFamily: Inter
```
    </CodeGroup>
  </Tab>
</Tabs>

## User Permissions

Manage who can view, edit, or administer your space. Assign roles to users or teams.

<Columns cols={2}>
  <Card title="Viewer" icon="eye" horizontal>
    Read-only access to all pages.
  </Card>
  <Card title="Editor" icon="edit-3" horizontal>
    Create and update documentation.
  </Card>
  <Card title="Admin" icon="shield" horizontal>
    Full control including permissions and exports.
  </Card>
</Columns>

Use this table to compare permissions:

| Role    | View Pages | Edit Pages | Manage Users | Export Content |
|---------|------------|------------|--------------|----------------|
| Viewer  | ✅        | ❌         | ❌           | ❌             |
| Editor  | ✅        | ✅         | ❌           | ✅             |
| Admin   | ✅        | ✅         | ✅           | ✅             |

<Expandable title="Add User Permissions via CLI" default-open="false">
  Run the following command to invite users programmatically.

````bash
janek-unon users invite --email="user@company.com" --role="editor" --space="your-space-id"
````

</Expandable>

## Export Options

Export your documentation as PDF, Markdown, or HTML for offline use or migration.

<Tabs>
  <Tab title="PDF Export" icon="file-text">
    Generates a printable version with your branding.

    ```json
    {
      "format": "pdf",
      "includeToc": true,
      "branding": true
    }
    ```
  </Tab>
  <Tab title="Markdown" icon="file">
    Exports raw MDX files for GitHub or other platforms.
  </Tab>
  <Tab title="HTML" icon="globe">
    Creates a static site bundle.
  </Tab>
</Tabs>

<Callout kind="alert">
  Exports include all public pages. Private pages require admin access.
</Callout>

<ExpandableGroup>
  <Expandable title="Advanced Export Settings">
    Customize exports with filters.

    ```javascript
    const exportConfig = {
      pages: ['/docs/*'],
      exclude: ['/private/*'],
      format: 'zip'
    };
    ```
  </Expandable>
</ExpandableGroup>
